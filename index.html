WSL-Specific

Built for WSL2

Kodra WSL handles the tricky parts of configuring a development environment inside WSL2.

⚙️ WSL Systemd

Enables systemd in WSL2 so Docker CE and other services start automatically on boot.

[boot]
systemd=true

🐳 Docker CE Auto-Start

Docker runs natively via wsl.conf — no Docker Desktop, no license costs, no overhead.

sudo service docker start

🔤 Windows Terminal Fonts

Installs JetBrains Mono Nerd Font and guides you through Windows Terminal configuration.

"fontFace": "JetBrainsMono Nerd Font"

🎨 Oh My Posh

Beautiful prompt theming that works perfectly in Windows Terminal with full glyph support.

eval "$(oh-my-posh init bash)"

💠 PowerShell 7

PowerShell 7 integration for cross-platform scripting alongside your Bash environment.

pwsh -Command "Get-AzContext"

🔗 Windows Integration

Seamless interop between WSL and Windows — open files, share paths, and access Windows apps.

explorer.exe . && code .
Simple Setup

How It Works

From fresh WSL2 to productive Azure development in three steps.

1

Open PowerShell → Run Command

Copy the install command and paste it into your WSL terminal. Kodra handles everything automatically.

wget -qO- kodra.wsl.codetocloud.io/boot.sh | bash
2

Tools Install Automatically

25+ tools are downloaded, installed, and configured. Docker, Azure CLI, kubectl, Oh My Posh — all wired together.

⏱ ~10 minutes on a typical connection
3

Verify → Start Building

Run kodra doctor to verify everything is working, then start building.

✓ Ready to go

Questions

Frequently Asked

Will it break my existing WSL setup? +

No. Kodra WSL backs up your existing shell configuration (.bashrc, .zshrc, .gitconfig) before making any changes. Your original configs are preserved and can be restored.

Can I customize what gets installed? +

The WSL edition installs a curated set of 25+ tools optimized for WSL2. It has fewer customization flags than the full Ubuntu edition, focusing on the most essential cloud-native tools for Windows developers.

Does it require Docker Desktop? +

No. Kodra WSL installs Docker CE that runs natively inside WSL2 via systemd. No Docker Desktop license needed — great for enterprise teams with licensing constraints.

What Windows versions are supported? +

Windows 11 with WSL2 enabled. You need Ubuntu installed from the Microsoft Store. Kodra WSL will check your WSL version during installation.

How is this different from the full Kodra? +

The full Kodra is designed for bare-metal Ubuntu 24.04 and includes desktop theming, Ghostty terminal, Starship prompt, and 40+ tools. Kodra WSL is tailored for WSL2 with Oh My Posh, Windows Terminal integration, and 25+ essential tools.

Is it free and open source? +

Yes! Kodra WSL is completely free under the MIT license. The code is on GitHub. Contributions are welcome!
